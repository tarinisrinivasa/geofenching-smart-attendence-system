Full A to Z Technical Project synopsis

IoT-Enabled Smart Attendance Management System with Real-Time Geofencing, Cryptographic WebAuthn Biometrics, and AI-Driven Bunking Anomaly Detection

1. Abstract Overview

Traditional academic and corporate attendance models—ranging from manual roll-calls to barcode card swipes and static QR scans—suffer from severe vulnerability to proxy check-ins (buddy system fraud), location spoofing, and unauthorized mid-session exits. To solve these problems, this project presents an end-to-end **Smart Attendance Management System** that combines high-precision GPS geolocation telemetry, hardware-bound device lockers, cryptographic WebAuthn biometrics, and active campus-wide geofencing to enforce absolute physical presence.

The system architecture is powered by a secure HTTPS Node.js/Express server paired with an SQLite relational database, providing distinct, modern dashboards for HODs, Class Coordinators, Teachers, and Students. The backend locks each student profile to a single unique hardware device_id on initial login, blocking account sharing. To check in, a student must pass a three-tiered validation process: matching GPS geofence bounds, scanning a dynamic HMAC SHA-256 rotating QR code (regenerated every 15 seconds), and completing a native fingerprint/FaceID scan via WebAuthn. Legacy keypad phone exemptions are supported via simulated SMS OTP gate overrides.

Following check-in, the student's mobile browser runs a continuous background telemetry script (Campus Guard) which continually calculates Haversine geofence bounds. If a student leaves the designated campus geofence perimeter during college hours without a valid digital out-pass granted by the HOD, the system immediately suspends check-in privileges and logs a geofence breach alert. At class completion, AI-driven backend analysis identifies unverified absences and consecutive-class skipping (bunking) anomalies, triggering parent SMS alerts.

2. A to Z Core Feature Breakdown

A. Cryptographic Authentication & Device Locks

  • Hardware ID Binding: Student logins are restricted to a single physical device identifier (device_id). Account-sharing attempts return a strict warning: "This account is registered on another device."
  • FIDO2 WebAuthn Biometrics: Leverages the browser's credential API to enforce fingerprint/FaceID verification directly at check-in, preventing proxy actions.
  • Fail-safe Password Fallback: Provides older smartphones with a secure login password confirmation layer combined with geofencing.

B. Geofenced Classroom Check-In & anti-cheat

  • Haversine Coordinates Check: Student GPS coordinates are validated against the class coordinates, taking device error margins into account.
  • Rotating HMAC QR Codes: Features a dynamically rotating QR code on the teacher portal that updates every 15 seconds, preventing students from taking photos and checking in from afar.
  • Keypad Phone SMS OTP Flows: Enables mobile-free keypad phone check-ins by generating random 4-digit codes simulated to an SMS gateway, which teachers or coordinators verify.

C. Background Telemetry (Campus Guard) & Penalties

  • Telemetry Verification: Initiates background geofence telemetry loops every 20 seconds.
  • Breach Lockouts: Leaving the campus boundaries without an active out-pass triggers a geofence breach alert, flags attendance_locked = 1, and blocks attendance check-ins.
  • Digital Out-Pass Lifecycle: Suspends tracking during permitted student leave windows.
  • Reversal Overrides: Connects breach alerts to the active class session, enabling HODs to reverse statuses to present.

D. AI-Driven Anomalies & Parents SMS Gateway

  • Unverified Absence Alerts: Ending a session auto-compiles unregistered students, logs alerts, and flags them.
  • AI Bunking Analysis: Detects consecutive-class skipping behaviors.
  • Parent Notification Gateway: Generates alert SMS notifications to registered parent contacts.

E. Web Application Firewall (WAF) & DDoS Protection Layer

  • Helmet Security Headers: Dynamically injects secure HTTP headers, configuring Content Security Policies (CSP) to block Cross-Site Scripting (XSS), script hijacking, and clickjacking.
  • Brute-Force Rate Limiting: Restricts request traffic parameters (500 requests/15 mins for general browsing; 15 requests/15 mins for sensitive login and verification routes) to prevent DDoS and credential stuffing.
  • Custom Request Sanitizer: Checks request payloads (body, query, params) against SQL Injection and Script Tag pattern signatures, automatically rejecting malicious requests with a 403 Forbidden alert.

3. Conclusion & System Integrity Summary

By consolidating multi-layered physical and cryptographic identity gates (GPS Geofences, Rotating HMAC tokens, WebAuthn keys) with a continuous client-side telemetry guard, this smart attendance management system eliminates proxy opportunities and provides administrators with a self-correcting validation system. The addition of legacy keypad phone modes, HOD pass controls, parent alerting, and consecutive skips analysis provides institutions with an all-inclusive framework that handles edge cases seamlessly while maintaining a modern user experience.
